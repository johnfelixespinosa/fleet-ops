The Problem

Fleet data is trapped behind complex queries and disconnected systems

An EV fleet generates thousands of data points — mileage, battery health, trip schedules, charging events, maintenance records, service center capabilities. Getting answers means writing queries, joining tables, and manually cross-referencing across systems. The data exists. Accessing it intelligently is the bottleneck.

MCP Tools

6 built-in tools — and growing

Each tool wraps a read-only database query behind the MCP protocol. New tools can be added by creating a Ruby class — the copilot discovers them automatically.

vehicles_due_for_maintenance
Projects daily mileage from recent trips, estimates when each vehicle will hit its threshold, ranks by urgency
upcoming_trips_for_vehicle
Returns scheduled trips with routes, distances, departure times, and cargo weights within a time window
service_centers_near_route
Haversine distance search along trip waypoints for EV-certified partner shops within radius
vehicle_health_summary
Aggregates efficiency trends, charging patterns, battery health, and maintenance history for a vehicle
draft_maintenance_recommendation
Combines vehicle + service center + trip into a structured recommendation with evidence and assumptions
fleet_query
General-purpose data lookup — vehicles by unit, trips by date, service centers by city, maintenance history
app/services/mcp/tools/vehicles_due_for_maintenance.rb 
module Mcp
  module Tools
    class VehiclesDueForMaintenance
      def self.tool_name = "vehicles_due_for_maintenance"

      def self.description
        "Find vehicles approaching or past their maintenance threshold
         within a given number of days."
      end

      def self.input_schema
        { type: "object", properties: {
            within_days: { type: "integer", default: 7 }
        }}
      end

      def self.execute(params)
        within_days = (params["within_days"] || 7).to_i
        vehicles = Vehicle.active

        results = vehicles.filter_map do |v|
          # Calculate avg daily mileage from last 30 days
          recent_trips = v.trips.where(status: "completed")
                                .where("return_at > ?", 30.days.ago)
          daily_miles = recent_trips.any? ? ... : 0

          projected = v.current_mileage + (daily_miles * within_days)
          miles_left = v.next_maintenance_due_mileage - v.current_mileage

          if projected >= v.next_maintenance_due_mileage || miles_left <= 1000
            { unit_number: v.unit_number, vehicle_id: v.id,
              miles_remaining: miles_left, maintenance_type: v.next_maintenance_type,
              daily_mileage_estimate: daily_miles, ... }
          end
        end

        results.sort_by { |r| r[:miles_remaining] }
      end
    end
  end
end

# Find Maintenance Opportunities

## Workflow

 1. Call `vehicles_due_for_maintenance` with the requested time window
 2. For each flagged vehicle, call `upcoming_trips_for_vehicle`
    to understand scheduling impact
 3. For vehicles with upcoming trips, call
    `service_centers_near_route` (leg: "return")
    to find low-disruption service options
 4. Rank opportunities by: miles remaining to threshold,
    number of affected trips, proximity of service centers
 5. Read `gotchas.md` before generating output
 6. Copy `templates/investigation-report.html`, fill with data
 7. Write to `/tmp/fleetops-reports/maintenance-investigation-{date}-{unique6}.html`
 8. Open in browser

# Fleet Safety Protocols
## Data Access
• All database access is READ-ONLY via curated MCP tools
• Never attempt to write, update, or delete operational data
• Never execute raw SQL or direct database commands
## Recommendations
• All recommendations are PROPOSALS for human review
• Always include evidence + assumptions
• Submitted as git PRs for fleet manager approval
## Audit
• Every tool call is logged automatically
• Every session is recorded for audit purposes
• Always identify as FleetOps Copilot in artifacts

Guardrails

Built-in safety at every step

Safety First
Before the copilot can access any fleet data, the safety protocols are automatically loaded. All data access is read-only — the copilot can query but never modify operational data.
Every Action Logged
Every tool call is recorded — which tool, what was queried, when, and by whom. When a session ends, the full audit trail is synced to the production dashboard.
Human Approval
Maintenance recommendations are proposals, not actions. They're submitted as GitHub PRs for fleet manager review. Every recommendation includes evidence and assumptions.

Observability

Copilot Session Tracking

Every copilot session is captured end-to-end and posted to the production dashboard. Fleet managers can see who used the copilot, what tools were called, what data was queried, and how long the session lasted.

❯ "Which charging session for EV-2501 last month was the most expensive?"
$95.14 — EA Station Modesto (en-route) on March 18. 177 kWh at 552 kW over 45 minutes. That's ~$0.54/kWh — roughly double depot rates, which is expected for DC fast charging ($0.45–$0.65/kWh vs $0.12–$0.18/kWh overnight).
❯ "What's the average charging cost for the eCascadia fleet in the last two weeks?"
Across 4 eCascadias (EV-2401, 2402, 2403, 2602): 37 sessions, $2,149 total, $58.08 avg/session. Depot charging at ~$0.14/kWh, en-route at ~$0.53/kWh. EV-2602 stands out with 5 en-route sessions — may need route optimization to reduce fast charging dependency.

What's Next

Endless possibilities — the platform is built to grow

Every new tool, skill, or integration is additive. The copilot's capabilities expand without rebuilding the core.

Access Control
SSH key authentication so only authorized users can start a copilot session. Each login is tied to a named identity in the audit trail.
Role-Based Tools
Dispatchers see trip and vehicle tools. Fleet managers get recommendation and approval tools. Mechanics only see their service briefs. Different roles, different capabilities.
Slack Notifications
"Send this maintenance report to the #fleet-ops channel." A new MCP tool that posts investigation results and recommendations directly to Slack.
Email Service Centers
"Email Bay Area Fleet Services — we're scheduling EV-2501 for a safety check next Thursday. Do you have availability?" Turn mock emails into real ones.
New Data Sources
Driver assignments, charger network availability, weather-adjusted routing, parts inventory. Each one is a new MCP tool — the copilot discovers it automatically.
Custom Skills
Compliance reporting, cost analysis workflows, battery degradation investigations — any repeatable investigation becomes a skill with a SKILL.md and templates.
